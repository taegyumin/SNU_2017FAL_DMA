데이터관리와 분석 2017-2 Dataset
IMLAB 박종혁


* 데이터

주어진 데이터는 사이트 A의 사용자, 게시물, 기타 관련 데이터입니다.
(데이터에 오류가 존재할 수 있으니, 발견시 즉시 알려주시기 바랍니다.)


* 데이터 형식
첫 줄은 데이터 어트리뷰트 정보, 그 이후는 모두 데이터입니다.
모든 줄은 comma(,)로 나뉘어있습니다.


* 데이터 파일

userInfo : 사용자 정보
posts : 전체 게시물 정보
questionPosts : 질문 게시물 정보
answerPosts : 답 게시물 정보
postHistory : 게시물 이력 정보
postLinks : 게시물 링크 정보
badges : 뱃지 정보
comments : 댓글 정보
tags : 태그 정보
votes : 'Vote' 정보

* 데이터 정보

 - userInfo
Id : 사용자 고유번호 (0 : 가상의 사용자)
Reputation : 평판
DisplayName : 사이트 내에서 사용되는 Name
Age : 사용자 나이 (빈 칸은, 가입 시 입력을 하지 않은 사용자)
CreationDate : 사용자 계정 생성 날짜시간
LastAccessDate : 마지막 계정 접속 날짜시간
WebsiteUrl
Location : 주소
AboutMe : 자기소개 글

 - posts
Id : 전체 게시물 고유번호 (0 : 가상의 게시물, 실제 존재하지 않음)
CreationDate : 전체 게시물 생성날짜
Body : 전체 게시물 내용
OwnerUserId : 게시물 작성자
LasActivityDate : 최근 게시물에서 마지막으로 활동이 일어난 날짜

 - questionPosts
Id : 질문 게시물 고유번호
PostId : 전체 게시물 고유번호
AcceptedAnswerId : 채택한 답 고유번호 (채택한 답이 없을 경우 : 0)
ViewCount : 조회수
Title : 제목
Tags : 태그

 - answerPosts (★★ 질문 게시물과 답 게시물 간 스키마 구성의 편의를 위하여, 가상의 Id = 0 인 게시물을 추가했습니다. 만약 id=0 이 출력되거나 포함되는 쿼리가 있다면, 이를 포함하지 말아야 합니다.)
Id : 답 게시물 고유번호 (0 : 가상의 답 게시물, 실제 존재하지 않음, 질문게시물 1의 채택되지 않은 답 게시물이지만, 여러 질문게시물의 채택 답 게시물이 될 수 있음)
PostId : 전체 게시물 고유번호
Accepted : 채택된 경우 1, 아닐 경우 2
ParentId : 질문 게시물 고유번호

 - postHistory
Id : 게시물 이력 고유번호
PostHistoryTypeId : 

1. Initial Title - The first title a question is asked with.
2. Initial Body - The first raw body text a post is submitted with.
3. Initial Tags - The first tags a question is asked with.
4. Edit Title - A question's title has been changed.
5. Edit Body - A post's body has been changed, the raw text is stored here as markdown.
6. Edit Tags - A question's tags have been changed.
7. Rollback Title - A question's title has reverted to a previous version.
8. Rollback Body - A post's body has reverted to a previous version - the raw text is stored here.
9. Rollback Tags - A question's tags have reverted to a previous version.
10. Post Closed - A post was voted to be closed.
11. Post Reopened - A post was voted to be reopened.
12. Post Deleted - A post was voted to be removed.
13. Post Undeleted - A post was voted to be restored.
14. Post Locked - A post was locked by a moderator.
15. Post Unlocked - A post was unlocked by a moderator.
16. Community Owned - A post has become community owned.
19. Question Protected - A question was protected by a moderator.

PostId : 전체 게시물 고유번호
CreationDate : 이력 변경 발생 날짜시간
UserInfoId : 변경한 사용자 고유번호
Text : 변경 내용
Comment : 덧글 정보

 - postLinks
Id : 링크 고유번호
CreationDate : 링크 발생 날짜 시간
PostId : 링크가 포함된 전체 게시물 고유번호
RelatedPostId : 링크의 전체 게시물 고유번호
LinkTypeId : 1이면 내용이 관련된 경우, 3이면 내용이 완전히 중복된 경우

 - badges : 
Id : 뱃지 획득이력 고유번호
UserInfoId : 뱃지 획득한 사용자 고유번호
Name : 획득한 뱃지 이름
Date : 뱃지 획득한 날짜시간

 - comments
Id : 댓글 고유번호
PostId : 댓글이 소속된 전체 게시물 고유번호
Score : 댓글이 획득한 점수
CreationDate : 댓글이 생성된 날짜시간
UserInfoId : 댓글을 생성한 사용자 고유번호

 - tags
Id : 태그 고유번호
TagName : 태그 이름
ExcerptPostId : 태그 요약문의 전체 게시물 고유번호
WikiPostId : 태그 백과사전의 전체 게시물 고유번호

 - votes
Id : Vote 고유번호
PostId : Vote가 발생한 전체 게시물 고유번호
VoteTypeId :
1: 질문자에 의해 채택됨
2: 좋아요
3: 싫어요
4: 글에 Offensive tag가 붙여짐 (hate speech, obscenities, abuse against people, or abuse of the community or system)
5: 즐겨찾기
6: 닫혀짐 (더 이상 변경 불가)
7: 닫힌 것이 다시 열림
8: 포상금 open
9: 포상금 종료
10: 삭제
11: 삭제 취소
15: 관리자 내용 검토
16: 제 3자의 게시문 글 수정 건의가 받아들여짐

CreationDate : Vote 발생 날짜 (날짜시간 아님)
UserInfoId : 즐겨찾기 (VoteTypeId = 5)를 한 사용자 고유번호, VoteType=5 가 아니면 null
BountyAmount : 포상금 종료(VoteTypeId = 9) 발생 시 수여된 포상금의 양, 포상금이 0(null)일 수도 있음. 이외의 VoteType에 대해서는 null
